# [NEW] Restricted Users

Starting in April 2026, we are migrating all SIM Management user to our new Role-Based User Management System.

This article is related to the new system. If your account’s users have not yet been migrated, please refer to the articlers concerning the [old user management system](managing-editing-user-accounts.md)

## What are Restricted users?

Restricted users are users that only have access to one or multiple Groups of SIMs.

For example, if you have created Groups in your account for each of your organization’s projects or customers, you might need to restrict some of your users so that they can only view and manage these specific groups.

These users can be easily identified, as their role will include “Restricted” in its name, and the label “Group” will be displayed next to it.

![image](/knowledge-center/assets/image-1.png)

## Creating a Restricted user

When selecting a Restricted role, you will see the down arrow below the account’s name.

Click on the down arrow to display the Groups and select which this new user should have access to

![image](/knowledge-center/assets/image-2.gif)

Managing a Restricted User

You can check the Groups the user has access in the Manager users menu.

The label will display how many accounts (folder icon) and groups (organigram icon) they have access to:

![image](/knowledge-center/assets/image-3.png)

Click on the label to display the detail

![image](/knowledge-center/assets/image-4.png)

You can add or remove groups to the Restricter users by using the “edit” button

![image](/knowledge-center/assets/image-5.png)

You can then add or remove Groups to this user’s permissions

![image](/knowledge-center/assets/image-6.png)
