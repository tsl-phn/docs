---
title: "POST /api/token — Obtain an Access Token"
sidebarTitle: "POST /api/token"
description: "Exchange your client_id and client_secret for an OAuth 2.0 Bearer token using the client_credentials grant type."
---

The `/api/token` endpoint is the single entry point for obtaining a Bearer token on the Transatel platform. You supply your `client_id` and `client_secret` — issued when your application was registered — and the platform returns a signed JWT that you attach to every subsequent API request. Tokens expire after one hour; when a token expires you simply request a new one using the same credentials.

## Endpoint

```
POST https://api.transatel.com/authentication/api/token
```

---

## Request

### Headers

<ParamField header="Authorization" type="string" required>
  Your credentials encoded as HTTP Basic Auth: `Basic <base64(client_id:client_secret)>`. Concatenate your `client_id` and `client_secret` with a colon, then Base64-encode the result.

  **Example:** if your `client_id` is `my-client` and your `client_secret` is `s3cr3t`, encode `my-client:s3cr3t` → `bXktY2xpZW50OnMzY3IzdA==`, then pass `Basic bXktY2xpZW50OnMzY3IzdA==`.
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/x-www-form-urlencoded`. Any other value causes a `400 invalid_request` error.
</ParamField>

### Body

<ParamField body="grant_type" type="string" required>
  Must be the literal string `client_credentials`. This is the only supported grant type. Any other value causes a `400 unsupported_grant_type` error.
</ParamField>

---

## Response

### 200 — Success

A successful request returns an `application/json` body containing your Bearer token.

<ResponseField name="access_token" type="string">
  A signed JWT Bearer token. Include this value in the `Authorization: Bearer <access_token>` header of every platform API request. The token is opaque to you — do not attempt to modify it.
</ResponseField>

<ResponseField name="expires_in" type="integer">
  The number of seconds until the token expires, measured from the moment of issuance. The platform currently issues tokens with a lifetime of `3600` seconds (one hour).
</ResponseField>

<ResponseField name="scope" type="string">
  A space-separated list of OAuth 2.0 scopes granted to this token (for example, `CONNECTIVITY_MANAGEMENT_READ CONNECTIVITY_MANAGEMENT_WRITE`). The scopes reflect the permissions associated with your client application.
</ResponseField>

<ResponseField name="token_type" type="string">
  Always `Bearer`. Use this value as the scheme prefix in the `Authorization` header of downstream API calls.
</ResponseField>

## Code examples

<CodeGroup>

```bash cURL
# Replace CLIENT_ID and CLIENT_SECRET with your actual credentials.
CLIENT_ID="your-client-id"
CLIENT_SECRET="your-client-secret"

# Base64-encode the credentials (colon-separated).
CREDENTIALS=$(echo -n "$CLIENT_ID:$CLIENT_SECRET" | base64)

curl --request POST \
  --url https://api.transatel.com/authentication/api/token \
  --header "Authorization: Basic $CREDENTIALS" \
  --header "Content-Type: application/x-www-form-urlencoded" \
  --data "grant_type=client_credentials"
```

```python Python
import base64
import urllib.request
import urllib.parse
import json

CLIENT_ID = "your-client-id"
CLIENT_SECRET = "your-client-secret"

# Encode credentials as Basic Auth.
credentials = base64.b64encode(f"{CLIENT_ID}:{CLIENT_SECRET}".encode()).decode()

url = "https://api.transatel.com/authentication/api/token"
data = urllib.parse.urlencode({"grant_type": "client_credentials"}).encode()

req = urllib.request.Request(
    url,
    data=data,
    headers={
        "Authorization": f"Basic {credentials}",
        "Content-Type": "application/x-www-form-urlencoded",
    },
    method="POST",
)

with urllib.request.urlopen(req) as response:
    token_data = json.loads(response.read())

access_token = token_data["access_token"]
print(f"Access token: {access_token}")
print(f"Expires in:   {token_data['expires_in']} seconds")
```

```javascript JavaScript
const CLIENT_ID = "your-client-id";
const CLIENT_SECRET = "your-client-secret";

// Base64-encode the credentials (works in Node.js and modern browsers).
const credentials = btoa(`${CLIENT_ID}:${CLIENT_SECRET}`);

const response = await fetch(
  "https://api.transatel.com/authentication/api/token",
  {
    method: "POST",
    headers: {
      Authorization: `Basic ${credentials}`,
      "Content-Type": "application/x-www-form-urlencoded",
    },
    body: new URLSearchParams({ grant_type: "client_credentials" }),
  }
);

if (!response.ok) {
  const error = await response.json();
  throw new Error(`${error.error}: ${error.error_description}`);
}

const { access_token, expires_in, scope, token_type } = await response.json();

console.log("Access token:", access_token);
console.log("Expires in:  ", expires_in, "seconds");
console.log("Scopes:      ", scope);
```

</CodeGroup>

## Token lifetime and refresh strategy

<Note>
  Tokens are valid for **3600 seconds (one hour)** from the moment they are issued. The `expires_in` field in the response tells you the exact lifetime.
</Note>

The Authentication API does not issue refresh tokens. When your token approaches expiration, request a fresh one by repeating the same `POST /api/token` call with your client credentials. We recommend the following strategy:

- **Cache the token** in your application for the duration of its lifetime rather than requesting a new one on every API call.
- **Refresh proactively** — treat the token as expired a short time before `expires_in` elapses (for example, 60 seconds early) to avoid race conditions at the boundary.
- **Handle `401` responses gracefully** on downstream API calls: if a protected endpoint returns `401 Unauthorized`, discard your cached token, request a new one, and retry the original request once.

<Tip>
  Store your `client_id` and `client_secret` in environment variables or a secrets manager — never hard-code them in source control. Rotate your credentials immediately if you suspect they have been exposed.
</Tip>

## Error responses

All errors follow a consistent JSON shape:

```json
{
  "error": "<error_code>",
  "error_description": "<human-readable explanation>",
  "error_uri": "https://api.transatel.com/authentication/docs/index.html#errors/<error_code>"
}
```

<Expandable title="400 — unsupported_grant_type">
  The `grant_type` field in the request body was present but its value is not `client_credentials`.

  **Resolution:** Set `grant_type=client_credentials` exactly as shown and ensure the body is URL-encoded.

  ```json
  {
    "error": "unsupported_grant_type",
    "error_description": "Unsupported grant type: password",
    "error_uri": "https://api.transatel.com/authentication/docs/index.html#errors/unsupported_grant_type"
  }
  ```
</Expandable>

<Expandable title="400 — invalid_request">
  One of the following conditions occurred:

  - The `Content-Type` header is not `application/x-www-form-urlencoded`.
  - The `grant_type` field is missing from the request body.
  - The `Authorization` header is absent or not in the correct `Basic <base64>` format.

  **Resolution:** Verify all three required inputs — the `Content-Type` header, the `Authorization` header, and the `grant_type` body field — are present and correctly formatted.

  ```json
  {
    "error": "invalid_request",
    "error_description": "Missing grant type",
    "error_uri": "https://api.transatel.com/authentication/docs/index.html#errors/invalid_request"
  }
  ```
</Expandable>

<Expandable title="401 — invalid_client">
  The `client_id` or `client_secret` decoded from the `Authorization` header did not match any registered client application.

  **Resolution:** Re-check your credentials in the Developer Portal. Ensure you are Base64-encoding `client_id:client_secret` (with a colon separator) and not the values individually.

  ```json
  {
    "error": "invalid_client",
    "error_description": "Bad credentials",
    "error_uri": "https://api.transatel.com/authentication/docs/index.html#errors/invalid_client"
  }
  ```
</Expandable>

<Expandable title="500 — server_error">
  An unexpected error occurred on the platform side. Your request was well-formed and your credentials are not at fault.

  **Resolution:** Retry the request using exponential back-off. If the error persists for more than a few minutes, contact Transatel Support and include the `error_uri` and timestamp from the response.

  ```json
  {
    "error": "server_error",
    "error_description": "An unexpected error has occurred",
    "error_uri": "https://api.transatel.com/authentication/docs/index.html#errors/server_error"
  }
  ```
</Expandable>
