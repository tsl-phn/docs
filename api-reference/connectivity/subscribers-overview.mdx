The Connectivity Management API gives you complete programmatic control over the subscriber lifecycle — from activating a SIM card to managing rate plans, handling SIM and MSISDN swaps, and orchestrating mobile number portability (MNP) across France, Belgium, and the United Kingdom. All endpoints are secured with OAuth 2.0 bearer tokens and scoped to either read or write permissions.

## Base URL

```
https://api.transatel.com/connectivity-management/subscribers
```

## Authentication

Every request must include a valid bearer token in the `Authorization` header:

```http
Authorization: Bearer <token>
```

Tokens carry one or both of the following scopes:

| Scope | Access |
|---|---|
| `CONNECTIVITY_MANAGEMENT_READ` | Read-only access to subscriber and SIM data |
| `CONNECTIVITY_MANAGEMENT_WRITE` | Write access for lifecycle and provisioning operations |

## Asynchronous Operations

<Note>
  Write operations (`POST`, `PUT`) return **HTTP 201** immediately with a `transactionId` and `transactionStatus: PENDING`. The actual provisioning happens asynchronously. Poll `GET /api/transactions/{transactionId}` until `asyncStatus` is `DONE` or `ERROR` before treating the operation as complete.
</Note>

See the [Transactions reference](/api-reference/connectivity/transactions) for full polling details.

## Common Error Codes

All endpoints share the following error response structure:

| HTTP Status | Code | Description |
|---|---|---|
| `400` | `BAD_REQUEST` | The request body or parameters are invalid or malformed |
| `403` | `INSUFFICIENT_PERMISSION` | Your token does not carry the required scope |
| `404` | `SUBSCRIBER_NOT_FOUND` | No subscriber exists for the given SIM serial or MSISDN |
| `404` | `TRANSACTION_ID_NOT_FOUND` | The provided transaction ID does not exist |
| `404` | `PORTABILITY_NOT_FOUND` | No portability request found for this subscriber |
| `409` | `CONFLICT` | Another provisioning request is already pending on this SIM |
| `500` | `INTERNAL_ERROR` | An unexpected server-side error occurred |
| `501` | `NOT_IMPLEMENTED` | The requested operation is not available for your account configuration |

## Subscriber Status Values

A subscriber moves through the following lifecycle states:

| Status | Description |
|---|---|
| `Available` | SIM is provisioned but not yet activated |
| `Loaded` | SIM is loaded and ready for activation |
| `TestMode` | SIM is in a sandbox test state |
| `Pre-Activated` | Activation has been initiated but is pending |
| `Active` | Subscriber is live and connected |
| `Suspended` | Network access is blocked; billing may still apply |
| `Tariff_Holiday` | Subscriber is in a billing pause period |
| `Operator_Barred` | SIM has been barred by the operator |
| `Terminated` | Subscriber has been permanently deactivated |
