The SMS Retrieve API gives you full visibility into both mobile-terminated (MT) messages you have sent and mobile-originated (MO) messages received from your on-net subscribers. Use the list endpoint to search across messages with rich filters — date ranges, delivery status, originator, destination, and more — then fetch a specific message by its `messageId` for full detail including delivery timestamps.

Both endpoints require a token with either the `NETWORK-SMS_SMS_READ` or `NETWORK-SMS_SMS_WRITE` scope.

**Base URL:** `https://api.transatel.com/network/sms`

---

## GET /api/sms

Returns a paginated list of SMS messages matching your filter criteria. Results are sorted by `creationDate` descending by default. All filter parameters support operator prefixes for flexible range and set queries.

### Filter Operators

Append an operator prefix to any filterable field value to control how the comparison is applied:

| Operator | Prefix   | Example                              | Meaning                      |
|----------|----------|--------------------------------------|------------------------------|
| Equal    | `eq:`    | `eq:2024-06-01T00:00:00Z`            | Exact match                  |
| Not equal | `neq:` | `neq:on-net`                         | Exclude value                |
| Greater than | `gt:` | `gt:2024-06-01T00:00:00Z`         | Strictly after               |
| Greater than or equal | `gte:` | `gte:2024-06-01T00:00:00Z` | On or after             |
| Less than | `lt:`  | `lt:2024-07-01T00:00:00Z`            | Strictly before              |
| Less than or equal | `lte:` | `lte:2024-07-01T00:00:00Z`    | On or before             |
| In set   | `in:`    | `in:delivered+expired`               | Matches any of (max 10, `+` separator) |
| Not in set | `notin:` | `notin:error+deleted`            | Excludes all listed values   |

Supply `creationDate` **twice** with `gte:` and `lte:` to define a time range.

### Query Parameters

<ParamField query="mvnoRef" type="string">
  Filter by account reference.
</ParamField>

<ParamField query="creationDate" type="string">
  ISO 8601 date-time with an operator prefix. Provide this parameter twice to create a range, e.g. `gte:2024-06-01T00:00:00Z` and `lte:2024-06-30T23:59:59Z`.
</ParamField>

<ParamField query="from" type="string">
  Filter by sender address (originator).
</ParamField>

<ParamField query="to" type="string">
  Filter by destination MSISDN.
</ParamField>

<ParamField query="network" type="string">
  `on-net` or `off-net`.
</ParamField>

<ParamField query="direction" type="string">
  `MO` (mobile-originated) or `MT` (mobile-terminated).
</ParamField>

<ParamField query="dlrStatus" type="string">
  Delivery report status. Accepted values: `delivered`, `expired`, `deleted`, `error`. Supports `in:` and `notin:` operators.
</ParamField>

<ParamField query="externalReference" type="string">
  Filter by your own reference string attached at send time.
</ParamField>

<ParamField query="page" type="integer">
  Page number. Range: `1–10`. Default: `1`.
</ParamField>

<ParamField query="size" type="integer">
  Results per page. Range: `1–100`. Default: `50`.
</ParamField>

<ParamField query="sort" type="string">
  Sort expression. Prefix field names with `+` (ascending) or `-` (descending). Default: `-creationDate`. Example: `+to+creationDate` sorts by destination ascending, then creation date ascending.
</ParamField>

### Response Fields

<ResponseField name="content" type="SmsMessage[]">
  Array of matching SMS message objects (see field descriptions below).
</ResponseField>

<ResponseField name="pageNumber" type="integer">
  The current page number returned.
</ResponseField>

<ResponseField name="totalPages" type="integer">
  Total number of pages available.
</ResponseField>

<ResponseField name="totalElements" type="integer">
  Total number of messages matching the filter criteria.
</ResponseField>

<CodeGroup>
```bash List delivered MT messages in June 2024
curl --request GET \
  --url "https://api.transatel.com/network/sms/api/sms?direction=MT&dlrStatus=delivered&creationDate=gte%3A2024-06-01T00%3A00%3A00Z&creationDate=lte%3A2024-06-30T23%3A59%3A59Z&size=50&page=1" \
  --header "Authorization: Bearer <token>"
```

```bash List MO messages from a specific number
curl --request GET \
  --url "https://api.transatel.com/network/sms/api/sms?direction=MO&from=33612345678&sort=-creationDate" \
  --header "Authorization: Bearer <token>"
```

## GET /api/sms/{messageId}

Retrieves a single SMS message by its unique identifier. This endpoint is the most reliable way to check current delivery status because it always returns the latest `dlrStatus` and `dlrDate`.

### Path Parameters

<ParamField path="messageId" type="string (UUID)" required>
  The message UUID returned when the message was submitted via `POST /api/sms`.
</ParamField>

### Response Fields — SmsMessage

<ResponseField name="messageId" type="string (UUID)">
  Unique message identifier.
</ResponseField>

<ResponseField name="mvnoRef" type="string">
  Your account reference.
</ResponseField>

<ResponseField name="creationDate" type="string">
  ISO 8601 creation timestamp.
</ResponseField>

<ResponseField name="createdBy" type="string">
  Identity of the caller that submitted or triggered the message.
</ResponseField>

<ResponseField name="direction" type="string">
  `MO` (received from subscriber) or `MT` (sent to subscriber).
</ResponseField>

<ResponseField name="from" type="string">
  Sender address.
</ResponseField>

<ResponseField name="to" type="string">
  Destination MSISDN.
</ResponseField>

<ResponseField name="type" type="string">
  `text` or `binary`.
</ResponseField>

<ResponseField name="encoding" type="string">
  `GSM-7`, `UCS-2`, or `BINARY`.
</ResponseField>

<ResponseField name="segments" type="integer">
  Number of SMS segments.
</ResponseField>

<ResponseField name="network" type="string">
  `on-net` or `off-net`.
</ResponseField>

<ResponseField name="validityPeriod" type="integer">
  Validity period in minutes. **MT messages only.**
</ResponseField>

<ResponseField name="dlr" type="boolean">
  Whether a delivery report was requested. **MT messages only.**
</ResponseField>

<ResponseField name="externalReference" type="string">
  Your reference string attached at send time. **MT messages only.**
</ResponseField>

<ResponseField name="text" type="string">
  Message text content (present when `type` is `text`).
</ResponseField>

<ResponseField name="binary" type="string">
  Hex-encoded binary payload (present when `type` is `binary`).
</ResponseField>

<ResponseField name="dlrDate" type="string">
  ISO 8601 timestamp of the delivery report event. **MT messages only, present when `dlr` is `true` and a report has been received.**
</ResponseField>

<ResponseField name="dlrStatus" type="string">
  Final delivery status: `delivered`, `expired`, `deleted`, or `error`. **MT messages only.**
</ResponseField>

### Error Responses

| HTTP Status | Error Code            | Meaning                                             |
|-------------|-----------------------|-----------------------------------------------------|
| 400         | `BAD_REQUEST`         | The `messageId` is not a valid UUID format.         |
| 404         | `MESSAGE_NOT_FOUND`   | No message with the given ID exists on your account.|
| 500         | `INTERNAL_ERROR`      | Unexpected server-side error.                       |
