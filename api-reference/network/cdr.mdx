The CDR API gives you programmatic access to the full usage event log for your subscriber base. Every data session, voice call, SMS, and MMS generates a Call Detail Record (CDR) that you can query, filter, and page through. Use CDRs for billing reconciliation, fraud detection, regulatory reporting, or customer-facing usage dashboards.

All requests require a bearer token with the `NETWORK-USAGE_CDR_READ` scope.

**Base URL:** `https://api.transatel.com/network/usage`

---

## GET /api/cdr

Returns a paginated list of CDR entries. Apply any combination of the filter parameters below to narrow results.

### Filter Operators

Append an operator prefix to a filter value to control the comparison type. For range fields (dates, byte counts, units) supply the parameter **twice** — once with `gte:` and once with `lte:`.

| Operator          | Prefix   | Example                              |
|-------------------|----------|--------------------------------------|
| Equal             | `eq:`    | `eq:CDR/DATA`                        |
| Not equal         | `neq:`   | `neq:CDR/VOICE`                      |
| Greater than      | `gt:`    | `gt:1000000`                         |
| Greater than or equal | `gte:` | `gte:2024-06-01T00:00:00Z`        |
| Less than         | `lt:`    | `lt:2024-07-01T00:00:00Z`            |
| Less than or equal | `lte:` | `lte:2024-06-30T23:59:59Z`           |
| In set            | `in:`    | `in:CDR/DATA+CDR/VOICE`              |
| Not in set        | `notin:` | `notin:error`                        |

### Pagination Parameters

<ParamField query="size" type="integer">
  Results per page. Range: `1–10000`. Default: `50`.
</ParamField>

<ParamField query="page" type="integer">
  Page number. Range: `1–200`. Default: `1`.
</ParamField>

<ParamField query="sort" type="string">
  Sort expression. Prefix with `-` (descending) or `+` (ascending). Default: `-eventDate`.
</ParamField>

### Event Filters

<ParamField query="eventId" type="string">
  Filter by specific event ID. Supports `eq:` operator only.
</ParamField>

<ParamField query="eventDate" type="string">
  ISO 8601 date-time. Provide twice with `gte:` and `lte:` to set a date range.
</ParamField>

<ParamField query="eventType" type="string">
  Type of usage event. Accepted values: `CDR/DATA`, `CDR/VOICE`, `CDR/SMS`, `CDR/MMS`. Supports `in:` and `notin:`.
</ParamField>

### Subscriber Filters

<ParamField query="msisdn" type="string">
  Filter by subscriber MSISDN.
</ParamField>

<ParamField query="imsi" type="string">
  Filter by subscriber IMSI.
</ParamField>

<ParamField query="simSerial" type="string">
  Filter by SIM card serial number (ICCID).
</ParamField>

<ParamField query="subscriberNumber" type="string">
  Filter by internal subscriber number.
</ParamField>

<ParamField query="externalReference" type="string">
  Filter by your custom subscriber reference.
</ParamField>

<ParamField query="mvnoRef" type="string">
  Filter by account reference.
</ParamField>

<ParamField query="group" type="string">
  Filter by subscriber group name.
</ParamField>

<ParamField query="imei" type="string">
  Filter by device IMEI.
</ParamField>

### Session Filters

<ParamField query="serviceType" type="string">
  Filter by service type code.
</ParamField>

<ParamField query="rat" type="integer">
  Radio Access Technology type (integer). See the [Data Session RAT type table](/api-reference/network/data-session#rat-type-reference) for values.
</ParamField>

<ParamField query="apn" type="string">
  Filter by Access Point Name (data CDRs only).
</ParamField>

<ParamField query="deviceIPv4" type="string">
  Filter by device IPv4 address (data CDRs only).
</ParamField>

<ParamField query="deviceIPv6" type="string">
  Filter by device IPv6 address (data CDRs only).
</ParamField>

<ParamField query="originCountry" type="string">
  Filter by the ISO country code where the session originated.
</ParamField>

<ParamField query="mcc" type="string">
  Filter by Mobile Country Code of the serving operator.
</ParamField>

<ParamField query="mnc" type="string">
  Filter by Mobile Network Code of the serving operator.
</ParamField>

<ParamField query="serviceOutcome" type="string">
  Filter by the final service outcome (e.g. `success`, `failure`).
</ParamField>

<ParamField query="switchingDomain" type="string">
  `PS` (Packet Switched) or `CS` (Circuit Switched). Applies to voice, SMS, and MMS CDRs.
</ParamField>

<ParamField query="aNumber" type="string">
  Calling party number (voice CDRs only).
</ParamField>

<ParamField query="bNumber" type="string">
  Called party number (voice CDRs only).
</ParamField>

<ParamField query="sessionId" type="string">
  Data session identifier.
</ParamField>

### Usage Volume Filters (data CDRs)

<ParamField query="downlink" type="integer">
  Downlink bytes. Supports `gte:`/`lte:` range.
</ParamField>

<ParamField query="uplink" type="integer">
  Uplink bytes. Supports `gte:`/`lte:` range.
</ParamField>

<ParamField query="total" type="integer">
  Total bytes (uplink + downlink). Supports `gte:`/`lte:` range.
</ParamField>

<ParamField query="usedServiceUnits" type="number">
  Used service units — seconds for voice, always `1` for SMS. Supports `gte:`/`lte:` range.
</ParamField>

<ParamField query="mmsSize" type="integer">
  MMS message size in bytes. Supports `gte:`/`lte:` range.
</ParamField>

<ResponseField name="content" type="CdrEntry[]">

```bash Data CDRs for a SIM, last 7 days
  --url "https://api.transatel.com/network/usage/api/cdr?simSerial=8933012345678901234&eventType=CDR%2FDATA&eventDate=gte%3A2024-06-24T00%3A00%3A00Z&eventDate=lte%3A2024-06-30T23%3A59%3A59Z&size=100&sort=-eventDate" \
  --header "Authorization: Bearer <token>"
```
